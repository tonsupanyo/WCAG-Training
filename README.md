# โครงการ การอบรมเชิงปฏิบัติการการสร้างเว็บไซต์ที่ทุกคนเข้าถึงได้ตามมาตรฐาน WCAG ขั้นพื้นฐานสำหรับนักพัฒนาเว็บไซต์

ยินดีต้อนรับสู่โปรเจกต์ต้นแบบระบบการจองตั๋วรถทัวร์ออนไลน์ **"ไทยทริป (Thai Trip)"** ซึ่งพัฒนาขึ้นเพื่อใช้เป็นสื่อกลางและกรณีศึกษาในการฝึกอบรมเชิงปฏิบัติการการพัฒนาเว็บไซต์ที่ทุกคนเข้าถึงได้ตามมาตรฐาน **Web Content Accessibility Guidelines (WCAG) 2.2 ระดับ AA**

โครงการนี้ร่วมพัฒนาโดย **ทีมพัฒนากรมผู้สูงอายุ** และ **บริษัท แมนพาว สตูดิโอ จำกัด**

---

## 💻 เทคโนโลยีที่ใช้งาน
ระบบนี้เขียนด้วยสถาปัตยกรรมเว็บไซต์แบบ Static (ไม่ต้องใช้ Server-side rendering) เพื่อการศึกษาโครงสร้างโค้ดพื้นฐานที่ชัดเจน:
- **HTML5**: โครงสร้างแบบ Semantic HTML และการใส่ ARIA Attributes เพื่อรองรับเครื่องมืออ่านหน้าจอ (Screen Reader)
- **Vanilla CSS**: ตกแต่งสไตล์แบบ Responsive รองรับการขยายขนาดข้อความและการแสดงผลในหลากหลายขนาดหน้าจอ
- **Vanilla JavaScript**: ควบคุมตรรกะการจองตั๋ว, การตรวจสอบฟอร์ม (Validation), ระบบจับเวลา และ Focus Management

---

## ♿ ฟีเจอร์เด่นด้านความเข้าถึง (Accessibility Features)
เว็บไซต์ต้นแบบ "ไทยทริป" มีการปรับปรุงตามเกณฑ์มาตรฐาน WCAG 2.2 AA ดังนี้:
- **Bypass Blocks (WCAG 2.4.1)**: มีลิงก์สำหรับข้ามไปยังเนื้อหาหลัก (Skip Link) ที่ส่วนบนสุดของทุกหน้าเพจ
- **Keyboard Navigation (WCAG 2.1.1)**: หน้าเว็บทุกส่วน หน้าต่าง Popup และระบบปฏิทิน สามารถควบคุมการใช้งานผ่านปุ่มคีย์บอร์ด (Tab, Shift+Tab, Enter, Space, Esc) ได้อย่างสมบูรณ์แบบโดยไม่เกิด Keyboard Trap
- **Focus Visible (WCAG 2.4.7)**: ออกแบบขอบเน้นสถานะปุ่มหรือช่องรับข้อมูลที่ได้รับโฟกัส (Focus Indicator) ชัดเจนด้วยสีตัดกันสูง
- **Contrast Minimum (WCAG 1.4.3 & 1.4.11)**: สีตัวอักษร สีปุ่ม และไอคอนต่าง ๆ มีอัตราส่วนความต่างสี (Contrast Ratio) ที่ผ่านเกณฑ์ไม่น้อยกว่า 4.5:1 สำหรับตัวหนังสือปกติ และ 3:1 สำหรับปุ่ม/ไอคอน
- **Reflow & Text Resize (WCAG 1.4.10 & 1.4.4)**: หน้าเว็บยืดหยุ่นรองรับการ Zoom ได้ถึง 200% โดยที่ข้อมูลไม่ตกหล่น และรองรับหน้าจอสมาร์ทโฟนขนาดเล็กสุดที่ความกว้าง 320px
- **Input Assistance (WCAG 3.3.1, 3.3.2, 3.3.3)**: แบบฟอร์มข้อมูลผู้โดยสารมีคำแนะนำประกอบที่ชัดเจน แจ้งเตือนข้อผิดพลาดด้วยคำอธิบายที่เป็นข้อความ และแนะนำแนวทางการแก้ไขที่ถูกต้อง

## 📁 โครงสร้างโฟลเดอร์ของโปรเจกต์
โปรเจกต์นี้ได้รับการปรับปรุงและจัดระเบียบไฟล์ดังนี้:

```
wcag-training/
├── css/
│   └── styles.css             # ไฟล์สไตล์หลักของเว็บไซต์ (อัปเดตค่าความต่างสีและโฟกัส)
├── js/
│   ├── common.js              # สคริปต์กลาง (ระบบจับเวลา/คำเตือน, การควบคุมการขยายตัวอักษร)
│   ├── index.js               # ควบคุมหน้าค้นหาเที่ยวรถ
│   ├── select-trip.js         # ควบคุมหน้าเลือกเที่ยวรถทัวร์
│   ├── select-seat.js         # ควบคุมหน้าแผนผังการเลือกที่นั่ง (Focus Management)
│   ├── passenger-info.js      # ควบคุมหน้ากรอกข้อมูลและตรวจสอบความถูกต้องฟอร์ม
│   ├── payment.js             # ควบคุมหน้าขั้นตอนการชำระเงินและข้อความแจ้งเตือน
│   └── booking-success.js     # ควบคุมหน้าแสดงตั๋วเดินทางอิเล็กทรอนิกส์
├── images/                    # รูปภาพประกอบและภาพไอคอนของระบบ
├── result/                    # รายงานผลการตรวจสอบ (Audit Reports) หน้าเพจต่าง ๆ (ดูรายละเอียดใน result/README.md)
├── index.html                 # หน้าแรก (ค้นหาเที่ยวรถ)
├── select-trip.html           # หน้าเลือกเที่ยวรถ
├── select-seat.html           # หน้าเลือกที่นั่ง
├── passenger-info.html        # หน้ากรอกข้อมูลผู้โดยสาร
├── payment.html               # หน้าชำระเงิน
└── booking-success.html       # หน้าทำรายการสำเร็จ
```

สามารถดับเบิ้ลคลิกเปิดไฟล์ [index.html](index.html) ผ่านเว็บบราวเซอร์ (เช่น Chrome, Safari, Firefox) เพื่อเริ่มต้นใช้งานและทดสอบระบบได้ทันที
